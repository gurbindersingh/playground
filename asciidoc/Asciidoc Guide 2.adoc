== Document Header & Metadata

Every AsciiDoc document can begin with a header block that defines the title, author, revision info, and document-wide attributes.

=== Document Title

The document title uses a single `=` sign:

----
= My Document Title
----

=== Author & Revision Lines

----
= Document Title
Author Name <author@example.com>
v1.2, 2026-03-22: Final revision
----

Multiple authors are separated by semicolons:

----
= Document Title
First Author <first@example.com>; Second Author <second@example.com>
----

=== Document Attributes

Attributes are key-value pairs set in the header with `:attribute-name: value`:

----
= My Document
:author: Jane Doe
:email: jane@example.com
:revnumber: 2.0
:revdate: 2026-03-22
:revremark: Major update
:description: A sample document
:keywords: asciidoc, guide, reference
:toc: left
:sectnums:
:icons: font
:imagesdir: ./images
:source-highlighter: highlight.js
----

To unset an attribute, use a `!` suffix or prefix:

----
:sectnums!:
:!sectnums:
----

Reference attributes in text with `{attribute-name}`:

----
This document was written by {author} on {revdate}.
----

=== Custom Attributes

----
:project-name: SuperWidget
:version: 3.4.1

Welcome to {project-name} version {version}.
----

== Section Titles (Headings)

Sections are defined by the number of `=` signs. AsciiDoc supports six heading levels:

----
= Document Title (Level 0)

== Level 1 Section

=== Level 2 Section

==== Level 3 Section

===== Level 4 Section

====== Level 5 Section
----

=== Discrete Headings

A discrete heading is not part of the section hierarchy:

----
[discrete]
== This Heading Won't Appear in the TOC
----

=== Custom Section IDs

----
[#my-custom-id]
== Section with Custom ID

Link to it: <<my-custom-id,click here>>
----

== Paragraphs

Paragraphs are simply blocks of text separated by blank lines.

----
This is the first paragraph.

This is the second paragraph.
Adjacent lines are joined
into a single paragraph.
----

=== Line Breaks

A hard line break within a paragraph uses `+` at the end of the line:

----
First line +
Second line +
Third line
----

Or set the `hardbreaks` option on a paragraph:

----
[%hardbreaks]
First line
Second line
Third line
----

To enable document-wide hard line breaks:

----
:hardbreaks-option:
----

=== Lead Paragraph

----
[.lead]
This paragraph will be rendered with a larger font as an introductory paragraph.
----

== Text Formatting

=== Inline Formatting

[cols="2,3", options="header"]
|===
| Syntax | Result

| `+*bold*+`
| *bold*

| `+_italic_+`
| _italic_

| `+*_bold italic_*+`
| *_bold italic_*

| `+` `monospace` `+`
| `monospace`

| `+*`bold monospace`*+`
| *`bold monospace`*

| `+#highlighted#+`
| #highlighted#

| `+[.underline]#underlined#+`
| [.underline]#underlined#

| `+[.line-through]#strikethrough#+`
| [.line-through]#strikethrough#

| `+[.overline]#overlined#+`
| [.overline]#overlined#

| `+^superscript^+`
| ^superscript^

| `+~subscript~+`
| ~subscript~

| `+"`curly quotes`"+`
| "`curly quotes`"
|===

=== Constrained vs. Unconstrained

Constrained formatting applies at word boundaries:

----
A *bold* word.
----

Unconstrained formatting (doubled markers) applies mid-word:

----
Fan**tast**ic
It__al__ics
----

=== Literal (Passthrough) Inline

Prevent inline substitutions with a passthrough:

----
+This *text* is not formatted+
----

Or use the inline passthrough macro:

----
pass:[This <em>text</em> is passed through as-is]
----

== Lists

=== Unordered Lists

----
* Item 1
* Item 2
** Nested item 2a
** Nested item 2b
*** Deeply nested
* Item 3
----

Alternative markers for visual distinction:

----
- Item 1
- Item 2
----

=== Ordered Lists

----
. First item
. Second item
.. Nested item a
.. Nested item b
... Deeply nested
. Third item
----

Explicit numbering styles:

----
[arabic]
. One
. Two

[loweralpha]
. a
. b

[upperalpha]
. A
. B

[lowerroman]
. i
. ii

[upperroman]
. I
. II
----

=== Starting Number

----
[start=5]
. This starts at 5
. This is 6
----

=== Reversed Lists

----
[%reversed]
. Third
. Second
. First
----

=== Checklist (Task List)

----
* [*] Completed task
* [x] Also completed
* [ ] Incomplete task
* Normal list item
----

=== Description Lists (Definition Lists)

----
Term 1:: Definition 1
Term 2:: Definition 2
Term 3::
  A longer definition
  that spans multiple lines.

Nested Term::: Nested definition
----

Horizontal layout:

----
[horizontal]
CPU:: Central Processing Unit
RAM:: Random Access Memory
SSD:: Solid State Drive
----

=== Question & Answer Lists

----
[qanda]
What is AsciiDoc?:: A text document format for writing documentation.
Why use it?:: It's simple, powerful, and converts to many formats.
----

=== Mixed Lists

----
Operating Systems::
  * Linux
  * macOS
  * Windows
Editors::
  . VS Code
  . Vim
  . Emacs
----

=== List Continuation

Use `+` on a line by itself to attach additional blocks to a list item:

----
* Item one
+
This paragraph belongs to item one.
+
----
This code block also belongs to item one.
----

* Item two
----

Or use an open block for complex content:

----
* Item with complex content
+
--
This paragraph is inside the list item.

Another paragraph still in the same list item.

 $ command-example
--
----

=== Open Block

An open block has no default styling and is often used as a container:

----

----

____

----
> To be or not to be,
> that is the question.
> -- William Shakespeare, Hamlet
----

----

|===
| Name | Age | City
| Alice | 30 | London
| Bob | 25 | Paris
|===
----

The first row is automatically a header if followed by a blank line:

----
|===
| Name | Age | City

| Alice | 30 | London

|===
----

=== Column Formatting

----
[cols="2,1,3"]
|===
| Wide column | Narrow | Widest column
| content | content | content
|===
----

Column alignment:

----
[cols="<,^,>"]
|===
| Left | Center | Right
| aligned | aligned | aligned
|===
----

Column styles:

----
[cols="1h,2,2"]
|===
| Header col | Normal | Normal
| Row header | data | data
|===
----

Full column specifiers:

----
[cols="1,2a,1e,1m,1s,1l"]
|===
| Default | AsciiDoc | Emphasis | Mono | Strong | Literal
|===
----

* `a` = AsciiDoc (process content as AsciiDoc)
* `e` = emphasis (italic)
* `m` = monospace
* `s` = strong (bold)
* `l` = literal
* `h` = header style

=== Cell Spanning

Column span:

----
|===
| Col 1 | Col 2 | Col 3

3+| This cell spans 3 columns

| A | B | C
|===
----

Row span:

----
|===
| Col 1 | Col 2

.2+| Spans 2 rows
| Row 1 Col 2
| Row 2 Col 2
|===
----

Both:

----
|===
| Col 1 | Col 2 | Col 3

2.3+| Spans 2 cols and 3 rows
| B
| C
| D
|===
----

=== Cell-Level Formatting

----
|===
| Normal | Aligned

>| Right-aligned cell
^| Centered cell

e| Emphasized cell
m| Monospace cell
s| Strong cell
a| *AsciiDoc* processed cell
|===
----

== Substitutions

=== Substitution Types

[cols="1,2", options="header"]
|===
| Name | Description
| `specialcharacters` | Escapes `<`, `>`, `&`
| `attributes` | Replaces `{attribute}` references
| `quotes` | Applies text formatting (`*bold*`, `_italic_`, etc.)
| `replacements` | Typographic replacements (arrows, dashes, etc.)

== Conditional Directives

=== ifdef / ifndef

----
\ifdef::backend-html5[]
This content only appears in HTML output.
\endif::[]

\ifndef::production[]
DRAFT — not for distribution.
\endif::[]
----

=== ifeval

----
\ifeval::[{sectnums} == true]
Section numbers are enabled.
\endif::[]

\ifeval::["{backend}" == "html5"]
HTML-specific content.
\endif::[]
----

Operators: `==`, `!=`, `<`, `>`, `<=`, `>=`

=== ifdef with Multiple Attributes

----
\ifdef::attr1,attr2[]     <-- OR: either attribute set
Content shown if attr1 OR attr2 is defined.
\endif::[]

\ifdef::attr1+attr2[]     <-- AND: both attributes set
Content shown if attr1 AND attr2 are both defined.
\endif::[]
----

=== Backslash Escaping

----

\https://not-a-link.com

\<< not a cross-reference>>
----

=== Passthrough

Inline:

----
pass:[*not bold*]

+*also not bold*+
----

Triple-plus (no substitutions at all):

----
+++<u>underlined in HTML</u>+++
----

Block:

----
++++
<div>
  Raw HTML output.
</div>
++++
----

== Docinfo (Header/Footer Injection)

Include custom HTML in the document head or body:

----
:docinfo: shared
----

Create files:

* `docinfo.html` — injected into `<head>`
* `docinfo-footer.html` — injected before `</body>`
* `docinfo-header.html` — injected after `<body>` opens (some renderers)

For private (per-document) docinfo:

----
:docinfo: private
----

Uses `<docname>-docinfo.html` and `<docname>-docinfo-footer.html`.

== Processing Commands

=== Asciidoctor CLI

----
# Basic HTML conversion
asciidoctor document.adoc

# Specify output file
asciidoctor -o output.html document.adoc

# Specify backend
asciidoctor -b docbook document.adoc

# Set attributes
asciidoctor -a toc=left -a sectnums document.adoc

# PDF output (requires asciidoctor-pdf)
asciidoctor-pdf document.adoc

# EPUB output (requires asciidoctor-epub3)
asciidoctor-epub3 document.adoc

# Reveal.js slides
asciidoctor-revealjs presentation.adoc

# Safe mode
asciidoctor --safe-mode safe document.adoc
----

=== Safe Modes

[cols="1,2", options="header"]
|===
| Mode | Description
| `unsafe` | No restrictions (default for CLI)
| `safe` | Disables include and some macros
| `server` | Like safe, also restricts paths
| `secure` | Maximum restrictions
|===

== Extensions and Diagrams

=== Asciidoctor Diagram

Embed diagrams from plain text using asciidoctor-diagram:

----
[plantuml, diagram-sequence, png]
....
@startuml
Alice -> Bob : Hello
Bob -> Alice : Hi there!
@enduml
....
----

----
[ditaa, diagram-boxes]
....
    +--------+   +--------+
    |  cGRE  |-->|  cBLU  |
    | Green  |   |  Blue  |
    +--------+   +--------+
....
----

----
[mermaid]
....
graph TD
    A[Start] --> B{Decision}
    B -->|Yes| C[Action 1]
    B -->|No| D[Action 2]
....
----

----
[graphviz, dot-example, svg]
....
digraph G {
    rankdir=LR
    A -> B -> C
    B -> D
}
....
----

== Tips and Best Practices

=== One Sentence Per Line

Write each sentence on its own line for better version control diffs:

----
This is the first sentence.
This is the second sentence.
They will be joined into a single paragraph.
----

=== File Organization

For large documents, split into multiple files:

----
= Main Document
\include::chapters/introduction.adoc[]
\include::chapters/getting-started.adoc[]
\include::chapters/advanced-topics.adoc[]
\include::chapters/appendix.adoc[]
----

=== Conditional Content by Output

----
\ifdef::backend-html5[]
++++
<script>console.log("HTML only");</script>
++++
\endif::[]

\ifdef::backend-pdf[]
NOTE: This only appears in PDF output.
\endif::[]
----

=== Portable Image Sizing

----
:imagesdir: images

// Relative to content width
image::photo.jpg[width=50%]

// Absolute in pixels
image::photo.jpg[width=400]
----

=== Prevent Unwanted Substitution

If you're documenting AsciiDoc itself, escape or passthrough the syntax:

----
To make text bold, wrap it in +*asterisks*+.

The pass:[include::] directive imports other files.
----

=== Custom CSS

----
:stylesheet: my-styles.css
:linkcss:
----

Or embed inline:

----
ifdef::backend-html5[]
++++
<style>
.custom-class { color: navy; font-weight: bold; }
</style>
++++
endif::[]
----
